# Project Kickoff Checklist

## Pre-Kickoff Preparation

### Project Definition
- [ ] Project name and description defined
- [ ] Business case documented
- [ ] Stakeholders identified
- [ ] Success criteria established
- [ ] High-level scope outlined

### Team Setup
- [ ] Project team assembled
- [ ] Roles and responsibilities defined
- [ ] Communication channels created
- [ ] Meeting cadence established

### Infrastructure
- [ ] Repository created
- [ ] CI/CD pipeline configured
- [ ] Development environments set up
- [ ] Access permissions granted

---

## Discovery Phase Setup

### Documentation
- [ ] CLAUDE.md customized for project
- [ ] Project README created
- [ ] Documentation structure established

### Requirements
- [ ] Requirements gathering sessions scheduled
- [ ] Stakeholder interviews planned
- [ ] User research approach defined
- [ ] Requirements document template ready

---

## Technical Setup

### Development Environment
- [ ] Language/framework versions specified
- [ ] Dependencies installed
- [ ] Local development environment works
- [ ] Environment variables documented

### Version Control
- [ ] Git repository initialized
- [ ] Branch strategy defined
- [ ] Branch protection rules configured
- [ ] .gitignore configured

### Code Quality
- [ ] Linter configured (ESLint/Pylint/etc.)
- [ ] Code formatter configured (Prettier/Black)
- [ ] Pre-commit hooks set up
- [ ] Code style guide reviewed

### Testing
- [ ] Test framework configured
- [ ] Test directory structure created
- [ ] Coverage thresholds defined
- [ ] CI test runs configured

---

## Claude Code Setup

### Configuration
- [ ] CLAUDE.md populated with project info
- [ ] `.claude/settings.json` configured
- [ ] Skills reviewed and customized
- [ ] Rules reviewed and customized

### MCP Servers
- [ ] Required MCP servers identified
- [ ] MCP servers configured in `.mcp.json`
- [ ] API keys/credentials set up (in environment)

---

## Communication

### Meetings
- [ ] Kickoff meeting scheduled
- [ ] Regular standup time agreed
- [ ] Sprint/iteration cadence defined
- [ ] Review/demo sessions planned

### Channels
- [ ] Team chat channel created
- [ ] Email distribution list set up
- [ ] Status reporting process defined
- [ ] Escalation path documented

---

## Kickoff Meeting Agenda

### Opening (10 min)
- [ ] Welcome and introductions
- [ ] Meeting objectives

### Project Overview (20 min)
- [ ] Business context and goals
- [ ] Success criteria
- [ ] Timeline overview

### Team & Process (15 min)
- [ ] Team introductions
- [ ] Roles and responsibilities
- [ ] Development process overview

### Technical Overview (15 min)
- [ ] Architecture overview
- [ ] Technology stack
- [ ] Key technical decisions

### Next Steps (10 min)
- [ ] Immediate priorities
- [ ] Action items
- [ ] Questions

---

## Post-Kickoff Tasks

### Immediate (Week 1)
- [ ] Meeting notes distributed
- [ ] Action items assigned
- [ ] First sprint/iteration planned
- [ ] Team started on discovery

### Short-term (Week 2-3)
- [ ] Discovery phase progressing
- [ ] Architecture taking shape
- [ ] Risks identified
- [ ] Dependencies documented
